# 界面设计与交互流程

## 核心交互理念

比赛进行中，手机横屏握持，屏幕分为**左右两大操作区域**：

```
      左手拇指                   右手拇指
      ┌────────┐               ┌────────┐
      │  上滑   │               │  上滑   │
      │  A队+1  │               │  B队+1  │
      │        │               │        │
      │  下滑   │               │  下滑   │
      │  A队-1  │               │  B队-1  │
      └────────┘               └────────┘
```

**为什么要这样做？**

| 传统按钮方式 | 手势滑动方式 |
|-------------|-------------|
| 需要低头找按钮 | 凭肌肉记忆滑动 |
| 手指定位精确 | 整个半屏都是触摸区域 |
| 容易误触相邻按钮 | 只在有意识滑动时触发 |
| 运动时操作困难 | 运动中也能盲操 |
| 两者手需要同一区域 | 左右手各管各队，不打架 |

## 设备方向策略

```
竖屏模式：                   横屏模式（比赛中强制）：
┌──────────────┐            ┌─────────────────────────────┐
│              │            │  ┌───────┐        ┌───────┐  │
│   设置页     │            │  │ 左半屏 │  比分  │ 右半屏 │  │
│   历史页     │            │  │ A队区域│  显示  │ B队区域│  │
│   首页       │            │  │  滑动  │  (中)  │  滑动  │  │
│              │            │  └───────┘        └───────┘  │
│  竖屏使用   │            │        横屏使用（比赛中）      │
└──────────────┘            └─────────────────────────────┘
```

- **非比赛页面**：竖屏，标准交互
- **进入计分页面**：自动切换为强制横屏
- **比赛结束**：可以切回竖屏查看结果

## 手势交互详解

### 滑动区域划分

```
┌──────────────────┬┬──────────────────┐
│   A队滑动区域     ││   B队滑动区域     │
│                  ││                  │
│   ┌────────────┐ ││ ┌────────────┐   │
│   │ 向上滑 +1  │ ││ │ 向上滑 +1  │   │
│   │            │ ││ │            │   │
│   │  👆        │ ││ │        👆  │   │
│   │  ── 中线   │ ││ │  ── 中线   │   │
│   │  👇        │ ││ │        👇  │   │
│   │            │ ││ │            │   │
│   │ 向下滑 -1  │ ││ │ 向下滑 -1  │   │
│   └────────────┘ ││ └────────────┘   │
│                  ││                  │
│  ← 全区域响应    ││  全区域响应 →    │
│    不划也能点    ││    不划也能点    │
└──────────────────┴┴──────────────────┘
```

### 手势规则

| 手势 | 触发条件 | 效果 |
|------|---------|------|
| 👆 **向上滑动** | 任一半屏内，滑动距离 ≥ 30dp | 该队 +1 分 |
| 👇 **向下滑动** | 任一半屏内，滑动距离 ≥ 30dp | 该队 -1 分 |
| 👆👆 **快速连续上滑** | 0.5 秒内连续 2 次 | 该队 +2 分（篮球快捷操作） |
| 👈 **左滑**（左侧边缘） | 从最左侧向右滑 | 撤销上一步 |
| 👉 **右滑**（右侧边缘） | 从最右侧向左滑 | 撤销上一步 |
| 🤏 **长按**（3 秒） | 任意位置长按 | 弹出菜单（结束/暂停/分享） |

### 视觉反馈

```
每次滑动得分时：

1. 手指滑动 → 跟随手指出现拖拽痕迹
2. 松手触发 → 
   ├── 分数数字弹跳动画（scale 1.0 → 1.3 → 1.0，200ms）
   ├── 对应半屏短暂高亮（A队红色闪一下，B队蓝色闪一下）
   ├── 手机短振动（15ms）
   └── 可选：屏幕上飘出 "+1" 文字消散动画
```

## 手势冲突处理优先级

```
触摸事件到达屏幕

     │
     ▼
┌──────────────┐
│ 1. 边缘滑动    │ ← 屏幕左/右边缘 20dp 内 → 撤销手势
│    （最高优先） │
└──────┬───────┘
       │ 不匹配
       ▼
┌──────────────┐
│ 2. 垂直滑动    │ ← 上下滑动 ≥ 30dp → 计分
│    （核心）    │
└──────┬───────┘
       │ 不匹配
       ▼
┌──────────────┐
│ 3. 长按       │ ← 按住不动 ≥ 3 秒 → 菜单
└──────┬───────┘
       │ 不匹配
       ▼
┌──────────────┐
│ 4. 忽略       │ ← 轻触、短滑等不触发任何操作
└──────────────┘
```

## 篮球/足球的特殊手势

排球每次 +1，篮球需要支持 +1/+2/+3：

```
排球（默认）：
  上滑 = +1
  下滑 = -1

篮球模式：
  单指上滑     = +1 分（罚球）
  双指上滑     = +2 分（两分球）
  三指上滑     = +3 分（三分球）  ← 技术亮点！
  下滑         = -1 分

足球模式：
  上滑 = +1
  下滑 = -1
```

## UI 设计要点

| 要点 | 规则 |
|------|------|
| 触摸区域 | 左右半屏整块区域都可以滑动，无精确瞄准要求 |
| 滑动阈值 | 滑动 30dp 即触发，响应灵敏但不误触 |
| 色彩区分 | A队（左半屏）红色主题，B队（右半屏）蓝色主题 |
| 强制横屏 | 进入计分页自动锁横屏，旋转 180° 可换边 |
| 防误触 | 下滑需要确认阈值，轻触不触发；结束比赛需二次确认 |
| 触觉反馈 | 每次计分短振动（15ms），减分长振动（30ms） |
| 字体 | 分数数字 64sp+ 醒目大字，窄屏也能看清 |
| 对比度 | 深色背景 + 高亮分数，阳光下可读 |
| 横屏优化 | 横屏时隐藏系统状态栏，全屏沉浸式 |
