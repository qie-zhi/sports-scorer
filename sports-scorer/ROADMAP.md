# 开发路线图

## 总览

```
第 1 阶段 ──→ 第 2 阶段 ──→ 第 3 阶段
 [MVP]        [增强]        [打磨]
 1-2 周       1 周          1 周
```

---

## 第 1 阶段：MVP 最小可行产品（1-2 周）

> 目标：可以完成一场完整的计分

### 里程碑 1.1：项目搭建（Day 1）

- [ ] 安装 Node.js、Android Studio、JDK
- [ ] `npx react-native init SportsScorer --template react-native-template-typescript`
- [ ] 配置 ESLint + Prettier
- [ ] 安装核心依赖（Zustand、React Navigation、AsyncStorage）
- [ ] 运行 Hello World 确认环境正常
- [ ] 创建项目目录结构

### 里程碑 1.2：核心计分逻辑（Day 2-3）

- [ ] 实现 Match 数据模型
- [ ] 实现 useMatch Hook（状态管理）
- [ ] 实现 addScore / subtractScore 函数
- [ ] 实现胜负判定逻辑
- [ ] 单元测试覆盖核心逻辑

### 里程碑 1.3：基础 UI（Day 4-5）

- [ ] ScoreBoard 计分板组件
- [ ] ScoreButton 计分按钮组件
- [ ] 运动选择页面
- [ ] 计分主界面布局
- [ ] 得分动画效果（数字弹出）

### 里程碑 1.4：持久化（Day 6-7）

- [ ] 比赛数据存储到 AsyncStorage
- [ ] 应用重启后恢复比赛状态
- [ ] 比赛结束自动保存

### ✅ 第 1 阶段验收标准
- 可以创建一场排球比赛
- 可以给两队加分和减分
- 达到 25 分自动判断赢得该局
- 先赢 3 局自动结束比赛
- 杀进程后重开，比赛数据还在

---

## 第 2 阶段：增强功能（1 周）

> 目标：支持多运动且体验完整

### 里程碑 2.1：计分规则系统（Day 1-2）

- [ ] 实现 SportConfig 配置系统
- [ ] 排球 25 分制完整实现
- [ ] 篮球无上限计分
- [ ] 足球计分

### 里程碑 2.2：撤销功能（Day 3）

- [ ] 实现 ScoreRecord 记录每次操作
- [ ] 实现撤销 Hook
- [ ] 撤销 UI 按钮
- [ ] 支持连续撤销

### 里程碑 2.3：比赛历史（Day 4-5）

- [ ] 历史列表页面
- [ ] 比赛详情查看
- [ ] 删除历史记录
- [ ] 空状态展示

### ✅ 第 2 阶段验收标准
- 排球按 25 分制正常运转（领先 2 分）
- 篮球可以按 1/2/3 分加分
- 误操作可以撤回
- 能看到历史比赛

---

## 第 3 阶段：分享与打磨（1 周）

> 目标：可分享结果，体验精致

### 里程碑 3.1：数据分享（Day 1-2）

- [ ] JSON 文件导出
- [ ] 调用系统分享面板（微信/QQ/蓝牙等）
- [ ] JSON 文件导入恢复比赛
- [ ] 分享内容格式化（比分卡片）

### 里程碑 3.2：UI 打磨（Day 3-4）

- [ ] 运动图标和颜色主题
- [ ] 过渡动画
- [ ] 深色模式支持
- [ ] 横屏优化

### 里程碑 3.3：健壮性（Day 5）

- [ ] 边界情况测试
- [ ] 空状态、加载状态、错误状态
- [ ] 性能优化
- [ ] 最终 Bug 修复

### ✅ 第 3 阶段验收标准
- 可以导出比赛为 JSON 文件
- 可以通过分享面板发送给朋友
- 朋友导入 JSON 可以查看比赛结果
- UI 在横竖屏下都正常显示

## 未来扩展（v2.0+）

| 功能 | 难度 | 说明 |
|------|------|------|
| 实时多人同步 | 高 | 蓝牙/WiFi Direct 两台设备同步计分 |
| 球员统计 | 中 | 记录每个球员的得分、失误 |
| 语音播报 | 低 | 语音播报当前比分 |
| 比赛视频标记 | 中 | 记录得分时间点，方便回看录像 |
| Apple Watch 伴侣 | 中 | 手表上也能计分 |
| 联赛模式 | 高 | 多场比赛积分排名 |
