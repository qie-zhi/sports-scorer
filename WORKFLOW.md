# Claude Code 常见工作流程

## 工作流 1：从零开始新项目

```
步骤 1: /init → 初始化项目的 CLAUDE.md
步骤 2: 描述项目需求 → Claude 生成初始代码结构
步骤 3: 确认结构 → Claude 逐步实现各模块
步骤 4: 编写测试 → 验证功能正确性
步骤 5: 审查代码 → /code-review 检查质量
步骤 6: 编写文档 → Claude 生成 README 和 API 文档
```

## 工作流 2：日常开发循环

```
     ┌──────────┐
     │ 开始编码  │
     └────┬─────┘
          │
          ▼
   ┌──────────────┐
   │ 向 Claude    │
   │ 描述任务     │
   └──────┬───────┘
          │
          ▼
   ┌──────────────┐
   │ Claude 生成  │
   │ 或修改代码   │
   └──────┬───────┘
          │
          ▼
   ┌──────────────┐
   │ 运行测试     │──── 失败 ──→ 告诉 Claude 错误信息，修复
   └──────┬───────┘
          │ 通过
          ▼
   ┌──────────────┐
   │ 人工审查     │──── 不满意 → 向 Claude 描述问题，调整
   └──────┬───────┘
          │ 满意
          ▼
   ┌──────────────┐
   │ Git Commit   │
   └──────────────┘
```

## 工作流 3：代码审查流程

```
步骤 1: git diff > changes.diff
步骤 2: 把 diff 发给 Claude 审查
步骤 3: 或用 /code-review 命令
步骤 4: Claude 指出问题 → 确认是否修复
步骤 5: 应用修复 → 再次审查
步骤 6: 合并代码
```

## 工作流 4：重构大项目

```
阶段 1: 理解（Understand）
  - 让 Claude 阅读项目结构和关键代码
  - 生成项目架构文档
  - 识别需要重构的模块

阶段 2: 计划（Plan）
  - 使用 EnterPlanMode 制定重构方案
  - 确定重构顺序和影响范围
  - 设定回滚策略

阶段 3: 执行（Execute）
  - 小步重构，每次改一个模块
  - 每次改动后运行测试
  - 出现问题立即 git reset

阶段 4: 验证（Verify）
  - 运行所有测试
  - 用 /verify 确认改动正确
  - 代码审查
```

## 工作流 5：调试工作流

```
1. 描述症状
   "应用在 [场景] 出现 [现象]"

2. Claude 分析
   让 Claude 读取相关代码文件

3. 添加日志
   Claude 在关键位置添加 console.log

4. 运行复现
   /run 运行应用，提供日志输出

5. 定位修复
   Claude 根据日志信息修复代码

6. 验证修复
   测试确认问题已解决

7. 清理日志
   移除调试用的日志代码
```

## 工作流 6：文档驱动开发

```
1. 先写 README.md
   描述你想构建什么、怎么用

2. 让 Claude 读 README
   Claude 基于文档生成代码框架

3. 逐步完善
   代码和文档交替完善

4. 文档作为验收标准
   代码实现应该和文档描述一致
```

## 使用技巧

- **善用 Git**：每个大改动前先 commit，方便回滚
- **分而治之**：大问题拆成小问题，逐个解决
- **保持沟通**：不断给 Claude 反馈，让它调整方向
- **验证优先**：让 Claude 先写测试，再写实现（TDD）
