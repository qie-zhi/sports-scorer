# 有效提示词编写指南（Prompt Engineering）

## 提示词的核心原则

### 1. 清晰明确（Clarity）

❌ **差的提示词：**
```
帮我改一下这个代码
```

✅ **好的提示词：**
```
请重构 `src/utils/parser.ts` 中的 parseUserData 函数，
把它拆分成更小的函数，每个函数只做一件事，
同时保持原有的 API 接口不变。
```

### 2. 提供上下文（Context）

❌ **差的提示词：**
```
加个功能
```

✅ **好的提示词：**
```
我正在开发一个博客系统（React + TypeScript + Express），
需要在文章列表页面添加一个搜索框，用户可以按标题或标签搜索文章。
搜索字段放在导航栏下方，文章列表上方。
```

### 3. 指定输出格式（Format）

❌ **差的提示词：**
```
写个排序函数
```

✅ **好的提示词：**
```
用 TypeScript 写一个冒泡排序函数，要求：
- 函数名：bubbleSort
- 参数：number[]
- 返回值：number[]（升序排列）
- 使用泛型，支持传入比较函数
- 添加 JSDoc 注释
```

### 4. 分步骤（Step by Step）

对于复杂任务，拆分为小步骤：

```
我想给项目添加用户认证功能。请按以下步骤帮我：
1. 先检查现有的代码结构和路由设置
2. 设计用户模型（用户名、邮箱、密码哈希）
3. 创建注册和登录的 API 端点
4. 添加 JWT 中间件
5. 在前端添加登录表单
每一步完成后请等我确认再继续下一步。
```

## Claude Code 特有的提示技巧

### 1. 指定文件路径

```
在 src/services/auth.ts 中添加 refreshToken 方法
```

### 2. 引用已有代码

```
参考 src/utils/helpers.ts 中的 formatDate 函数的风格，
为 formatCurrency 函数添加类似的处理逻辑
```

### 3. 约束条件

```
帮我优化 src/components/DataTable.tsx 的性能，
但不要改变任何用户可见的行为和样式。
不要引入新的依赖库。
```

### 4. 对话式迭代

```
上次你创建的登录功能运行良好，
不过我想加一个"记住我"的复选框，用 localStorage 存储 token。
请只修改登录页面组件，不要改后端。
```

## 常见模式

| 模式 | 示例 |
|------|------|
| **解释代码** | `解释 src/app.ts 中第 45-78 行的逻辑` |
| **修复 Bug** | `当用户提交空表单时应用崩溃了，帮我找到并修复这个问题` |
| **添加测试** | `为 src/utils/validators.ts 中的所有函数编写 Jest 单元测试` |
| **重构代码** | `把 src/components/Header.tsx 拆分成多个小组件` |
| **迁移代码** | `把 src/api/ 中的回调风格改成 async/await` |
| **生成文档** | `为 src/lib/database.ts 生成 API 文档` |
